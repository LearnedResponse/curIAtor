# Completed public backlog items

Retired public work-orders, kept for provenance. The live backlog should describe work that still
needs a local implementation, release action, or external operating step; completed work belongs here
so shipped examples do not keep looking unstarted.

## When an item retires here

An item lands here when its scoped work-order is **fully delivered and verified**. Follow-on ideas stay
inside the file as roadmap notes, but they do not keep the original work-order live unless they are part
of that scope.

## Convention

1. Move the file here unchanged.
2. Move its index line in [`../README.md`](../README.md) to the **## Shipped** list, appending the
   collection or runner commit that proves the shipped state.
3. Update active backlog links to point at `completed/<item>.md` when they cite the retired provenance.
